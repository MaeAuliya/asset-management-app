# Product Strategy Roadmap

## Product Summary
This product is a modern mobile-based office asset lending application designed to manage borrowing and returning office assets in a structured and trackable way. The scope is intentionally focused on two categories: `Laptop` and `Projector`.

The product exists to replace a manual borrowing process that causes schedule confusion, poor asset visibility, and missed returns. The app prioritizes operational clarity, real-time availability, and a strong reminder and alert system.

Approved default UI design reference is Google Stitch project `Asset Catalog` (`projects/13780564782865433795`). That project should guide visual and interaction decisions for user-facing work until replaced by a newer approved Stitch project.

## Business Goal
- Digitize office asset borrowing and return workflows
- Reduce overdue returns
- Improve visibility of asset availability
- Speed up borrowing and return actions with barcode-based interaction
- Give admins a practical monitoring and approval console on mobile

## Current Pain Points
1. Borrowing and return schedules are disorganized.
2. Real-time asset availability is difficult to track.
3. There is no reminder system for returning borrowed assets.

## Target Users

### Admin
- Office staff responsible for managing assets and lending operations
- Needs fast visibility into asset status, pending approvals, overdue items, and urgent demand
- Needs tools for barcode generation, asset management, monitoring, and exports

### User
- Office employee borrowing assets for work, meetings, or training
- Needs a quick way to find available assets, borrow them, return them, and receive reminders

## Asset Scope
- `Laptop`
- `Projector`

## Product Principles
- Mobile-first and fast to use in real operational contexts
- Professional, practical, and easy to learn
- Notification-centered for reminders and urgent cases
- Clear status communication on every relevant screen
- Designed for efficient office administration, not consumer-style browsing
- Barcode-first for user navigation and task completion
- Request-first for admin operational navigation

## Core Business Rules
- Single-item borrowing can be completed by scanning the physical barcode attached to the asset.
- Bulk borrowing requires admin approval.
- Bulk borrowing uses a barcode generated by admin.
- Laptop borrowing often supports training sessions and medium-to-large scale requests.
- Projector borrowing supports general office and event use.
- Admin can configure borrowing time limits dynamically.

## Product Capabilities

### User Capabilities
- Login
- Logout
- Borrow asset
- Return asset
- Receive return reminder notifications
- View borrowing history
- View notifications
- Open barcode scan directly from primary app navigation

### Admin Capabilities
- CRUD users
- CRUD assets
- View borrowing statistics for today and all-time
- View borrowing history
- Export history data
- Print barcode
- Configure dynamic borrowing duration
- Approve bulk borrowing requests
- Receive urgent notifications for high-priority asset shortages or overdue situations
- Access scan-related actions from dashboard and asset workflows without replacing primary request navigation

## Technical Direction
- Flutter mobile application built on the existing Clean Architecture template foundation
- Existing template dependencies remain the baseline technology stack
- Supabase is the planned platform for:
  - primary relational data storage
  - authentication and role mapping
  - real-time asset availability and request updates
  - notification event orchestration planning

## MVP Roadmap

### MVP
- Authentication and role-based entry
- User dashboard
- Admin dashboard
- Asset list and asset detail
- Single-item borrow flow with barcode scan
- Return flow
- Borrowing history and detail
- Notification center for reminders and alerts
- Admin asset management
- Admin user management
- Bulk borrowing approval workflow
- Dynamic borrowing duration settings
- Basic today/all-time statistics

### Approved IA Baseline
- User shell: `Home`, `Assets`, `Scan`, `History`, `Settings or More`
- Notifications are still required, but are entered from badge, banner, and secondary entry points instead of a primary tab
- Admin shell: `Dashboard`, `Assets`, `Requests`, `History`, `Settings`
- Admin scan remains a quick action, not a primary navigation tab

### Phase 2
- Export history flows
- Barcode generation and print workflow refinement
- Stronger dashboard analytics
- Better overdue escalation handling
- Bulk operations for admin asset management

### Later
- More advanced alert prioritization
- Deeper operational insights by department or event
- Expanded notification rules and automation
- Optional integration with office hardware or inventory systems

## Success Metrics
- Borrow and return completion rate
- Overdue rate reduction
- Reminder open-to-action rate
- Approval turnaround time for bulk requests
- Time needed to identify available assets
- Admin response time to urgent alerts

## Product Outcome
The app should become the default office workflow for borrowing and returning laptops and projectors, replacing scattered manual coordination with a clear mobile system centered on availability, reminders, and admin visibility.
